## 1. 游戏核心机制概览

### 游戏类型
**叠牌**是一款融合了扑克手牌识别、物理塔楼叠放与Roguelike卡牌构筑的策略游戏。

### 核心游戏循环

```
关卡开始 → 玩家摆牌阶段 → 计分阶段 → 关卡结束
               ↓
         (每关3次计分机会，每次1次弃牌机会)
```

### 棋盘结构

```
┌─────────────────────────────────────────────────────┐
│  第0层（顶层）: [ 槽位1 ]                    [强化槽] │ ← 1个扑克槽
│  第1层（中层）: [ 槽位1 ] [ 槽位2 ]          [强化槽] │ ← 2个扑克槽
│  第2层（底层）: [ 槽位1 ] [ 槽位2 ] [ 槽位3 ] [强化槽] │ ← 3个扑克槽
└─────────────────────────────────────────────────────┘
```

**承重规则**：上层所有牌面值总和 **必须 <** 该层所有牌面值总和，否则触发坍塌。

## 2. 数值体系设计

### 2.1 扑克牌面值

| 牌面 | 数值 | 说明 |
|------|------|------|
| 2    | 2    | 最轻的牌，适合放在顶层 |
| 3    | 3    |      |
| 4    | 4    |      |
| 5    | 5    |      |
| 6    | 6    |      |
| 7    | 7    |      |
| 8    | 8    |      |
| 9    | 9    |      |
| 10   | 10   |      |
| J    | 11   | 皇室牌，高分但承重大 |
| Q    | 12   | 皇室牌，高分但承重大 |
| K    | 13   | 皇室牌，高分但承重大 |
| A    | 14   | 最重/最高分；可作低顺（A-2-3）使用 |

**设计说明**：
- 数值 = 承重值 = 基础计分值，三者统一，保持简洁
- 高数值牌（J/Q/K/A）带来更高基础分，但会增加结构承重风险
- A在顺子判定中可作 1 使用（A-2-3 低顺），增加策略灵活性

---

### 2.2 牌型倍率

| 牌型 | 倍率 | 触发条件 | 槽位需求 |
|------|------|---------|---------|
| 单张（Single）    | ×1.0 | 1张或无法构成其他牌型 | 任意层 |
| 对子（Pair）      | ×2.0 | 2张同点 | 顶层/中层 |
| 三条（Three of a Kind） | ×3.0 | 3张同点 | 底层 |
| 顺子（Straight）  | ×3.5 | 3张连续点数（含低顺A-2-3） | 底层 |
| 同花（Flush）     | ×4.0 | 3张同花色 | 底层 |
| 同花顺（Straight Flush） | ×6.0 | 3张同花色且连续 | 底层 |

**牌型层分布限制**：
- 顶层（1槽）：只能出单张
- 中层（2槽）：可出单张×2 或 对子
- 底层（3槽）：可出三条/顺子/同花/同花顺，或更低级组合

**设计说明**：
- 同花顺倍率是单张的6倍，创造清晰的风险/收益阶梯
- 顺子（×3.5）和同花（×4.0）的差异体现"数字连续"和"花色统一"的不同策略路线
- 三条（×3.0）略低于顺子，鼓励玩家追求更高组合

### 2.3 分数计算公式

#### 单层基础分

```
单层基础分 = Σ(所有检测到的子牌型) × Σ(子牌型中牌面值) × 对应倍率
```

#### 含强化牌的单层最终分

```
单层最终分 = (单层基础分 × 层倍率加成) + 层固定加成
```

其中：
- `层倍率加成` = 1.0（基础）+ 强化牌对本层的倍率修正之和
- `层固定加成` = 强化牌对本层的固定分加成之和

#### 总得分

```
本轮总分 = Σ(所有层最终分) × 挑战牌惩罚倍率
```

#### 计算示例

**场景**：底层放置 ♠7 ♥8 ♣9（顺子），安装了"顺子狂热"强化牌

```
基础分 = (7+8+9) × 3.5 = 24 × 3.5 = 84
层倍率 = 1.0（基础）+ 2.0（顺子狂热）= 3.0
最终分 = 84 × 3.0 = 252
```

**场景**：底层放置 ♥J ♥Q ♥K（同花），安装了"皇室专属"强化牌

```
基础分 = (11+12+13) × 4.0 = 36 × 4.0 = 144
层固定加成 = 3张皇室牌 × 50 = 150
最终分 = (144 × 1.0) + 150 = 294
```

### 2.4 承重系统

#### 基础规则

```
坍塌条件：某层上方所有层的牌面值总和 ≥ 该层牌面值总和
```

即：对于第 i 层，若 Σ(第0层到第i-1层所有牌面值) ≥ Σ(第i层所有牌面值)，则触发坍塌。

#### 承重计算示例

```
顶层（第0层）: A(14)         → 承重 = 14
中层（第1层）: K(13) + Q(12) → 承重 = 25
底层（第2层）: 5+6+7 = 18    → 承重 = 18

检查中层: 上方承重(14) < 本层承重(25) ✓ 安全
检查底层: 上方承重(14+25=39) > 本层承重(18) ✗ 坍塌！
```

#### 坍塌后果

- 触发坍塌的层及以上所有层的牌全部进入弃牌堆
- 不触发额外惩罚分（坍塌本身就是损失）

#### 基础承重（Foundation）

- 初始关卡：基础承重 = 无限大（第一关无坍塌风险）
- 后续关卡：上一关结算时场上存活牌的总牌面值作为下一关底层的基础承重
- 作用：模拟"地基"概念，让玩家早期规划影响后期游戏

### 2.5 关卡目标分数曲线

公式：`目标分 = floor(50 × 1.6^(关卡数-1))`

| 关卡 | 目标分 | 增长率 |
|------|--------|--------|
| 1    | 50     | —      |
| 2    | 80     | +60%   |
| 3    | 128    | +60%   |
| 4    | 205    | +60%   |
| 5    | 327    | +59%   |
| 6    | 524    | +60%   |
| 7    | 838    | +60%   |
| 8    | 1,342  | +60%   |
| 9    | 2,147  | +60%   |
| 10   | 3,436  | +60%   |

**最大理论分（无强化，纯底层同花顺）**：
```
底层 ♥A ♥K ♥Q → 同花顺
基础分 = (14+13+12) × 6.0 = 39 × 6.0 = 234

中层 ♠A ♥K → 对子（若同点）不太可能，用单张
中层 ♠J ♦J → 对子 = (11+11) × 2.0 = 44

顶层 ♣10 → 单张 = 10 × 1.0 = 10

每轮最高裸分约 ≈ 288（不含强化）
3次机会上限 ≈ 864（不含强化加成）
```

**设计意图**：
- 第1-3关：仅靠裸分即可通关，让玩家熟悉机制
- 第4-6关：需要强化牌加成才能稳定通关
- 第7+关：需要强化组合、精心构牌才能通关

## 3. 卡牌设计文档

### 卡牌通用属性

每张卡牌包含以下基础属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识符，格式：`{类型}_{英文名}` |
| `name` | string | 显示名称（中文） |
| `description` | string | 效果描述（玩家可见） |

### 3.1 强化牌（Enhance Cards）

强化牌安装在各层的强化槽中，提供持续性被动效果。每层1个强化槽，全场最多3张强化牌同时生效。

#### 强化牌设计原则

1. **层绑定**：效果通常只作用于安装层，不影响其他层
2. **被动触发**：通过监听 `SCORE_LAYER`（最常见）等事件自动触发
3. **可移动**：玩家可在摆牌阶段调整强化牌的安装层（成本：消耗一次操作）
4. **叠加性**：多张强化牌效果可在同一层叠加

#### 现有强化牌

---

##### `enhance_straight_fever` | 顺子狂热
- **稀有度**：稀有
- **商店价格**：5金
- **效果**：本层含顺子（或同花顺）时，本层计分乘区 +2.0
- **触发事件**：`SCORE_LAYER`
- **触发条件**：本层检测到顺子/同花顺牌型
- **数值**：`scoreMultiplier += 2.0`
- **强度评级**：★★★★☆

**设计说明**：
- 与顺子（×3.5）组合：基础倍率 3.5 × (1.0+2.0) = 10.5 倍
- 与同花顺（×6.0）组合：6.0 × 3.0 = 18 倍（全场最高）
- 鼓励玩家追求顺子相关牌型，增加构牌方向

**交互示例**：
```
底层 ♦5 ♦6 ♦7（同花顺）+ 顺子狂热
基础分 = (5+6+7) × 6.0 = 108
倍率加成 = 1.0 + 2.0 = 3.0
最终分 = 108 × 3.0 = 324
```

---

##### `enhance_hollow_brick` | 空心砖
- **稀有度**：稀有
- **商店价格**：4金
- **效果**：本层计分牌面值翻倍（×2.0），但该层在坍塌承重中视为 0
- **触发事件**：`SCORE_LAYER`
- **触发条件**：无条件触发
- **数值**：`scoreMultiplier *= 2.0`，`overrideLayerWeight = 0`
- **强度评级**：★★★★★（高风险高回报）

**设计说明**：
- 安装在顶层时风险最低（顶层本就无上层压力）
- 安装在中层时，顶层承重不再被中层承担，底层压力骤增
- 安装在底层时极危险：底层承重归零，必须依赖基础承重（Foundation）
- 是全场乘算倍率最高的强化牌，核心战略牌

**典型用法**：
```
顶层安装空心砖 → 顶层承重 = 0（顶层对下层无压力）
中层只需支撑自身 → 更容易放重牌
效果：顶层得分 ×2
```

---

##### `enhance_royal_exclusive` | 皇室专属
- **稀有度**：普通
- **商店价格**：3金
- **效果**：本层每有一张 J/Q/K，本层计分 +50（固定加成）
- **触发事件**：`SCORE_LAYER`
- **触发条件**：本层有 J(11)/Q(12)/K(13)
- **数值**：`scoreBonusFlat += royalCount × 50`（最多 +150）
- **强度评级**：★★★☆☆

**设计说明**：
- 加成为固定值，不受倍率影响（在倍率计算后加算）
- 底层3张皇室牌可获 +150 固定分
- J/Q/K牌面值高（11-13），本身承重大，该强化牌为使用皇室牌提供额外收益
- 与"空心砖"不可叠加（空心砖固定加成不会翻倍），玩家需权衡

**交互示例**：
```
底层 ♠J ♦Q ♣K（同花色？否，为同花需同色）
→ 检测为3张单牌或对子+单牌
→ 皇室专属加成：3 × 50 = +150
```

---

##### `enhance_lucky_draw` | 幸运摸牌
- **稀有度**：普通
- **商店价格**：3金
- **效果**：每次从牌库摸到 A，本关手牌上限 +1
- **触发事件**：`CARD_DRAWN`
- **触发条件**：摸到的牌为 A（rank=14）
- **数值**：`gameState.handSize += 1`（持续全关）
- **强度评级**：★★☆☆☆（概率依赖）

**设计说明**：
- 4张A触发4次：最多 +4 手牌上限
- 手牌上限增加意味着更多选择空间（更容易凑成好牌型）
- 与牌库中A的数量强相关（正常52张牌中有4张A）
- 适合追求手牌数量的构筑风格

**注意**：`_layerIndex`参数被忽略（`_`前缀表示未使用），该牌全局生效而非层绑定。

### 3.2 挑战牌（Challenge Cards）

挑战牌由游戏自动添加，为玩家带来负面效果。每关开始时出现，不可移除。

#### 挑战牌设计原则

1. **全局作用**：挑战牌不绑定层，影响全局
2. **高优先级**：触发优先级为 10（强化牌为 0），在强化效果后执行
3. **惩罚多样**：从分数惩罚、手牌惩罚到结构破坏，覆盖多个维度
4. **难度递进**：高关卡同时叠加多张挑战牌

---

##### `challenge_acid_rain` | 酸雨腐蚀
- **出现关卡**：第3关起（建议）
- **效果**：每次计分结束后，随机降低场上 3 张牌牌面值 -2
- **触发事件**：`SCORE_END`
- **数值**：`MODIFY_RANDOM_CARDS: count=3, valueChange=-2`，重算坍塌
- **惩罚强度**：★★★☆☆

**设计说明**：
- 连续累积效果：3次计分机会 = 最多降低9张次牌面值-2
- 会触发坍塌重算，可能导致结构崩塌
- 克制：保留底层高面值牌作为"缓冲"，或使用"混凝土"消耗牌补偿

---

##### `challenge_earthquake` | 强震
- **出现关卡**：第2关起（建议）
- **效果**：每次计分结束后，当前总得分 ×0.9（扣除10%）
- **触发事件**：`SCORE_END`
- **数值**：`MODIFY_TOTAL_SCORE: multiplier=0.9`
- **惩罚强度**：★★☆☆☆

**设计说明**：
- 3次计分机会：(1-0.1)^3 ≈ 累积约损失27%分数
- 鼓励玩家提高单次计分的得分（与多次小分相比，大分更抗惩罚）
- 对分数乘法加成影响最大（高倍率牌型受损更多）

---

##### `challenge_detention` | 扣留
- **出现关卡**：第4关起（建议）
- **效果**：每次计分结束后，本关手牌上限 -1
- **触发事件**：`SCORE_END`
- **数值**：`MODIFY_HAND_SIZE: delta=-1, trimExcess=true`
- **惩罚强度**：★★★★☆

**设计说明**：
- 起始手牌上限8，3次计分后：8→7→6（到第3次时减少到6）
- 最终手牌只剩6张，凑顺子/同花概率大幅下降
- 克制："幸运摸牌"强化牌可对冲（摸到A时+1）
- 对依赖大量手牌选择的玩家惩罚最重

---

##### `challenge_opening_curse` | 开局诅咒
- **出现关卡**：第5关起（建议）
- **效果**：关卡开始时，随机摧毁底层（第2层）一个槽位
- **触发事件**：`LEVEL_START`
- **数值**：`DESTROY_RANDOM_SLOT: layerIndex=2, count=1`
- **惩罚强度**：★★★★★

**设计说明**：
- 底层从3槽变为2槽，无法构成三条/顺子/同花/同花顺等高分牌型
- 每关开始就触发，无法提前应对
- 严重限制底层得分潜力
- 克制：提前在中层、顶层布置高效强化牌，减少对底层依赖

### 3.3 消耗牌（Consume Cards）

消耗牌为一次性使用，在摆牌阶段消耗，立即产生战术效果。

#### 消耗牌设计原则

1. **即时效果**：在摆牌阶段使用，立即触发 `sideEffects`
2. **一次性**：使用后消失
3. **战术价值**：解决特定困境，非通用万能卡
4. **需要目标**：部分消耗牌需选择目标牌或层

---

##### `consume_concrete` | 混凝土
- **商店价格**：3金
- **效果**：目标牌牌面值临时 +5（本次计分有效）
- **需要目标**：是（目标：场上某张牌）
- **数值**：`MODIFY_RANDOM_CARDS: valueChange=+5, recalculateCollapse=true`
- **战术价值**：★★★☆☆

**使用场景**：
- 底层牌面值不足以支撑上层时，临时加固底层某张牌
- 让某张牌的面值满足顺子条件（如把8变成13，配合Q和J）
- 增加某层的承重能力

**注意**：效果持续全局（不仅本轮），需谨慎使用——给高层牌加重可能导致结构不稳。

---

##### `consume_precision_blast` | 精准爆破
- **商店价格**：4金
- **效果**：摧毁目标牌，不触发坍塌
- **需要目标**：是（目标：场上某张牌）
- **数值**：`DESTROY_SPECIFIC_CARD: skipCollapse=true`
- **战术价值**：★★★★☆

**使用场景**：
- 移除高层中过重的牌，防止下层坍塌
- 清理不能构成有效牌型的"废牌"
- 与"空心砖"配合：先装空心砖，再移除多余牌

**与普通移除的区别**：不触发坍塌重算，即使移除后承重关系改变也不触发崩塌判定。

---

##### `consume_dark_deal` | 暗箱操作
- **商店价格**：5金
- **效果**：将选中的2张手牌替换为黑桃A（♠A）
- **需要目标**：否（消耗时自动选取手牌）
- **数值**：`REPLACE_HAND_CARDS: count=2, suit='spades', rank=14`
- **战术价值**：★★★★★（高风险高价值）

**使用场景**：
- 一次性获得2张 A（rank=14），快速构建高分单张
- 与"幸运摸牌"配合：摸牌阶段已经抽到的A不触发（这是REPLACE）——实际是替换手牌，不经过摸牌事件
- 代价：失去2张原有手牌，牺牲手牌多样性

**风险**：
- 两张黑桃A面值高（14），放在高层时承重压力极大
- 若无法有效利用高面值牌，会造成手牌浪费

---

##### `consume_jackscrew` | 千斤顶
- **商店价格**：3金
- **效果**：本次计分基层承重临时 +20
- **需要目标**：否
- **数值**：`TEMP_FOUNDATION_BONUS: bonus=20`（仅本轮）
- **战术价值**：★★★☆☆

### 6.2 基础承重继承机制

```
第N关结束时存活的所有牌面值之和 = 第N+1关的初始基础承重

示例：
第1关结束时底层 6+7+8 = 21（存活）
→ 第2关基础承重 = 21
→ 第2关底层至少需要牌面值 > 21（上层承重总和）才安全
```

**继承策略建议**：
- 保留低值牌在底层（减少遗留承重压力），但得分也低
- 保留高值牌可提升下关基础承重，但本关下层承重压力大

### 6.3 商店经济设计（建议）

每关结束进入商店，根据得分获得金币：

```
基础奖励 = 3金
得分奖励 = floor(得分 / 目标分 × 5)金（最多5金）
关卡奖励 = 关卡数金（第3关=3金）

总奖励 = 3 + 得分奖励 + 关卡奖励（建议上限12金/关）
```

**商店价格参考**：

| 物品类型 | 价格范围 | 说明 |
|---------|---------|------|
| 普通强化牌 | 2-3金 | 条件苛刻或效果一般 |
| 稀有强化牌 | 4-6金 | 高效但需技巧 |
| 史诗强化牌 | 7-9金 | 游戏改变级效果 |
| 普通消耗牌 | 2-3金 | 即用即弃 |
| 强力消耗牌 | 4-5金 | 高收益单次效果 |
| 跳过关卡奖励 | —    | 跳过商店可获 +2金补偿 |
